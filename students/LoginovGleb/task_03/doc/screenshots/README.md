# Инструкция по созданию скриншотов для отчета

Эта инструкция описывает как получить каждый скриншот, указанный в главном README.md

## Подготовка

1. Откройте приложение в браузере Chrome или Firefox
2. Откройте DevTools (F12)
3. Перейдите на вкладку Network
4. Убедитесь, что опция "Disable cache" выключена

---

## 01_interface.png - Интерфейс приложения

**Что показывает:** Главный интерфейс приложения с формой поиска и карточкой погоды

**Как получить:**

1. Откройте `src/index.html` в браузере
2. Введите название города (например, "Moscow")
3. Нажмите кнопку "Поиск"
4. Дождитесь загрузки данных
5. Сделайте скриншот всего окна приложения (Alt + PrtScr)
6. Обрежьте при необходимости, чтобы показать:
   - Заголовок приложения
   - Форму поиска с кнопками
   - Карточку с погодой
   - Популярные города внизу

## 02_retry_console.png - Retry в консоли

**Что показывает:** Логи повторных попыток запросов в консоли DevTools

**Как получить:**

1. Откройте DevTools (F12) → вкладка Console
2. Очистите консоль (Ctrl + L)
3. **Временно измените API ключ на неверный** в `src/scripts/weatherApi.js`:
   ```javascript
   const API_KEY = 'invalid_key_for_testing';
   ```
4. Введите название города и нажмите "Поиск"
5. В консоли появятся сообщения о попытках:
   - "Попытка 1 не удалась. Повтор через 500мс..."
   - "Попытка 2 не удалась. Повтор через 1000мс..."
6. Сделайте скриншот консоли с логами
7. **Верните правильный API ключ обратно**

## 03_first_request.png - Первый запрос (с сервера)

**Что показывает:** Карточка погоды с badge "🌐 Свежие данные"

**Как получить:**

1. Обновите страницу (F5) - это очистит кэш

6. Под поиском должно быть сообщение: "✨ Свежие данные загружены с сервера"
7. Сделайте скриншот карточки и информации о кэше

## 04_cached_request.png - Повторный запрос (из кэша)

**Что показывает:** Карточка погоды с badge "💾 Из кэша"

**Как получить:**

1. **НЕ обновляя страницу** после предыдущего шага
2. Введите тот же город (например, "London")
3. Нажмите "Поиск" снова
4. Убедитесь, что на карточке badge "💾 Из кэша"
5. Под поиском должно быть сообщение: "⏱️ Данные из кэша. Актуальны ещё XX сек."
6. Сделайте скриншот карточки и информации о кэше

## 05_network_first.png - DevTools Network - первый запрос

**Что показывает:** Вкладка Network с запросом к API (Status 200)

**Как получить:**

1. Откройте DevTools (F12) → вкладка Network
2. Убедитесь, что "Disable cache" выключен
3. Очистите список запросов (кнопка 🚫 или Ctrl + L)
4. Обновите страницу (F5)
5. Введите город и нажмите "Поиск"
6. В списке Network найдите запрос к `api.openweathermap.org`
7. Кликните на него, чтобы увидеть детали
8. Сделайте скриншот, который показывает:
   - Список запросов (статус 200, размер, время)
   - Вкладку Headers с URL
   - Response (опционально)

---

## 06_network_cached.png - DevTools Network - кэшированный запрос

**Что показывает:** Вкладка Network БЕЗ запроса (данные из in-memory кэша)

**Как получить:**

1. Продолжаем с предыдущего шага (НЕ обновляем страницу)
2. Очистите список запросов в Network (кнопка 🚫)
3. Введите **тот же город** и нажмите "Поиск"
4. Обратите внимание: **запроса к API не будет** (список пустой или только локальные ресурсы)
5. Сделайте скриншот пустой вкладки Network
6. В скриншоте должно быть видно:
   - Пустой список запросов (или без запросов к API)
   - Карточка погоды с badge "💾 Из кэша" на заднем плане

**Примечание:** Если используется HTTP-кэш (304), то запрос будет, но со статусом "304 Not Modified" или "(from disk cache)"

## 07_cache_indicator.png - Кнопка обновления и индикатор кэша

**Что показывает:** Информационный блок о кэше с таймером и кнопкой "Обновить сейчас"

**Как получить:**

1. Выполните поиск города
2. Сразу же повторите поиск того же города (данные из кэша)
3. Под формой поиска появится желтое уведомление:
   - "⏱️ Данные из кэша. Актуальны ещё XX сек."
   - Кнопка "Обновить сейчас"
4. Также видна кнопка "🔄 Обновить" рядом с кнопкой "Поиск"
5. Сделайте скриншот формы поиска с этим уведомлением

## 08_abort_controller.png - Отмена запроса в консоли

**Что показывает:** Логи отмены запросов при быстром вводе

**Как получить:**

1. Откройте DevTools (F12) → Console
2. Очистите консоль
3. **Быстро** введите несколько городов подряд (не дожидаясь загрузки):
   - Введите "Paris" → Enter
   - Сразу введите "Berlin" → Enter
   - Сразу введите "Tokyo" → Enter
4. В консоли появятся сообщения:
   - "Запрос был отменен" (для Paris и Berlin)
   - Успешная загрузка для Tokyo
5. Также можно увидеть в Network запросы со статусом "canceled"
6. Сделайте скриншот консоли с логами отмены

**Альтернативный способ:**

1. **Временно увеличьте таймаут** в `utils.js` до 10000 мс
2. Введите город и нажмите "Поиск"
3. **Сразу** введите другой город и нажмите "Поиск" снова
4. Первый запрос будет отменен
5. В консоли: "Запрос был отменен"

## Дополнительные скриншоты (опционально)

### 09_skeleton_loading.png — Skeleton (загрузка)

Минималистичный способ получить красивый скриншот состояния загрузки.

#### Способ A (рекомендуется): отдельная страница-предпросмотр

1. Откройте `src/skeleton-preview.html` в браузере
2. Установите ширину окна ~520–560px (по желанию, чтобы была одна карточка)
3. Снимите скриншот белой карточки скелетона
4. Сохраните как `doc/screenshots/09_skeleton_loading.png`

#### Способ B (в приложении)

1. Временно увеличьте таймаут в `utils.js` → `timeoutMs: 3000`
2. Откройте `src/index.html`, выполните поиск города
3. Пока идёт загрузка, снимите скриншот области карточки
4. Верните `timeoutMs` обратно (5000)

### error_state.png - Состояние ошибки

**Как получить:**

1. Введите несуществующий город (например, "asdfghjkl")
2. Нажмите "Поиск"
3. Появится карточка с ошибкой:
   - Иконка ⚠️
   - Заголовок "Ошибка"
   - Сообщение "Город ... не найден"
4. Сделайте скриншот

### mobile_responsive.png - Мобильная версия

**Как получить:**

1. Откройте DevTools (F12)
2. Включите Device Toolbar (Ctrl + Shift + M)
3. Выберите устройство (например, iPhone 12)
4. Убедитесь, что интерфейс адаптировался
5. Сделайте скриншот

## Финальные шаги

После создания всех скриншотов:

1. Сохраните их в папку `doc/скриншоты/` с соответствующими именами
2. Формат: PNG или JPG
3. Разрешение: достаточное для читаемости (рекомендуется 1920x1080)
4. Убедитесь, что все пути в `README.md` корректны
5. Проверьте, что скриншоты отображаются в README

## Советы

- Используйте инструмент "Snipping Tool" (Windows) или "Screenshot" (Mac)
- Для скриншотов DevTools можно использовать встроенную функцию (Ctrl + Shift + P → "Capture screenshot")
- Обрезайте скриншоты, чтобы показать только важную информацию
- Добавьте стрелки или выделения для акцента на важных элементах (опционально)
- Проверьте, что текст читаем даже после сжатия GitHub
